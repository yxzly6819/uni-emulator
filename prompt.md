# 《大学模拟器》Web 游戏开发 Prompt

你是一个资深全栈工程师，请根据以下设计文档实现一款名为《大学模拟器》的讽刺风格大学模拟经营游戏。技术栈使用 **React + TypeScript + Tailwind CSS**（通过 Vite 创建），状态管理使用 React Context + useReducer。要求 UI 精致，所有文本数据化以便于维护。

当前环境是wsl

---

## 一、游戏全局

### 1.1 游戏概念
- 类型：文字决策 / 模拟经营
- 时长：约20分钟
- 结构：4年共8个学期，每学期2次操作节点（期初、期中），共16次关键决策
- 可选专业：EECS（电子工程与计算机科学）、法学
- 核心：通过课程选择、精力分配和随机事件，模拟大学生活并走向不同结局，全程贯穿对当代高等教育的讽刺

### 1.2 核心循环
```
START → 专业选择 → 学期循环(8次) → 结局判定 → END
```
每个学期：
```
期初操作（选课 + 精力分配）→ 上半学期结算（含随机事件）→ 期中操作（调整精力分配）→ 下半学期结算（含随机事件）→ 期末总结 → 身心检查 → 下一学期
```

## 二、核心状态设计

使用一个全局 `GameState` 对象，通过 `useReducer` 管理。初始状态：

```ts
interface PlayerState {
  major: 'EECS' | 'LAW' | null;      // 专业，游戏开始时选择
  money: number;                      // 初始 3000
  gpaRecords: { courseId: string; gradePoint: number; term: number }[]; // 每门已修课程
  trueAbility: number;                // 隐藏，初始 20，界面不显示数值
  mindBody: number;                   // 0-100，初始 80
  currentYear: number;                // 1-4
  currentSemester: number;            // 1-8
  completedPolitics: string[];        // 已完成的政治课id，如 ['moral','history','mao','situation']
  flags: {
    hasLove: boolean;
    loveCostActive: boolean;          // 恋爱持续扣钱标记
    injured: boolean;                 // 本半学期无法竞赛
    jyyCompleted: boolean;            // 完成《操作系统》
    contestWon: boolean;              // 本半学期竞赛获奖（用于事件触发）
    selfstudyHard: boolean;           // 大四是否认真备考
    expelled: boolean;                // 被开除
  };
  eventHistory: string[];             // 已触发过的事件id，避免重复
  currentCourse: Course | null;       // 本学期所选课程
  termPhase: 'major_select' | 'semester_start' | 'mid_feedback' | 'semester_end' | 'ending';
}
```

额外辅助状态：`currentHalf: 1 | 2`（上半学期/下半学期），临时精力分配数据，事件临时效果等。

## 三、属性系统

### 3.1 可见属性
- 金钱：期初自动 +1500 生活费。兼职、事件等会改变。无上限。
- 绩点：每学期末结算一门课的绩点，四年累计 GPA = 算术平均（已修课程）
- 身心状态：0-100，初始 80。

### 3.2 隐藏属性
- 真实能力：不显示数值，只通过事件文字暗示。影响学习效率、竞赛成功率、部分事件选项解锁及成功率。

### 3.3 关键规则
- 身心 < 30 时，学习效率衰减为 **0.5**（乘在绩点产出上）
- 身心 ≤ 0 或一学期没有修任何课程（且未毕业），立即触发强制休学结局
- 真实能力对学习效率的系数：`0.5 + trueAbility / 200`，即能力 0 时效率 50%，100 时 100%

## 四、学期详细流程

### 4.1 期初：选课 + 上半学期精力分配

**选课**  
- 从可选课程池中选择本学期 **最多 1 门** 课。可选课程 = 专业课池中未修过的 + 剩余未修政治课（共4门：`moral`, `history`, `mao`, `situation`）。  
- 可主动选择“不选课”，此时 `currentCourse = null`，该学期仅进行额外活动。  
- 已修过的课不可再选。

**精力分配**  
精力点：10 点。  
- 若选了课，先选择投入档位：  
  - 逃课（0精力）、摸鱼（1）、正常（3）、认真（5）、死磕（7）  
- 剩余精力分配给额外活动（可多选，总消耗 ≤ 10）：  
  - 兼职 (3)、竞赛/项目 (5)、休息（不消耗，视为剩余精力闲置时自动恢复身心？这里改为消耗 0，但必须主动选择才能恢复身心）、自学 (4)  
  *注意：休息必须主动勾选并消耗 0 精力，但提供一个身心 +15 的效果（不可叠加，一次半学期只能选一次）*。  
- 如果上半学期有受伤标记（`flags.injured`），竞赛/项目不可选。  
- 毛概的“混脸熟”机制已取消，不再出现。

### 4.2 上半学期结算（期中反馈）

1. **课程效果**（如果有课）  
   - 基础绩点区间 = 课程定义的基础区间 × 真实能力系数  
   - 如果身心 < 30，再乘 0.5  
   - 点名惩罚：若档位为逃课，按课程点名率判定是否被点中。被点中则该课最终绩点 -0.3，身心 -5。  
   - 真实能力变化：按课程和档位增加 `trueAbility`。  
   - 身心消耗：档位基础消耗 + 课程特殊消耗。  
   - 特殊课程规则（如模拟电路绩点上限、科研训练完全看真实能力等）严格执行。  
   - 最终生成一个 **预期绩点范围** 字符串（如“3.0 - 3.5”），及身心、能力变化文本暗示。

2. **额外活动结算**  
   - 兼职：金钱 +800~1200（随机），若专业相关（EECS）有 30% 概率真实能力 +1  
   - 竞赛/项目：  
     - 判定成功：真实能力≥50 时 70% 成功率；20~49 时 40%；<20 时 10%  
     - 成功则金钱 +500~3000（视能力而定），真实能力 +4~8，设置 `contestWon = true`  
     - 失败则无奖金，真实能力 +1~3，身心 -5  
   - 休息：身心 +15（上限 100）  
   - 自学：真实能力 +3~6  

3. **身心状态更新**：限制在 0~100。若 ≤0 立即进入休学结局。

4. **触发上半学期随机事件**：根据条件选取 1 个事件，玩家做出选择后影响属性。事件结果可能包含精力扣除（`energyDeduct`），若扣除后该半学期精力不足，则额外扣身心。

5. **期中反馈界面**：展示预期绩点范围、能力变化暗示、金钱、身心、辛辣总结文字。

### 4.3 期中操作（下半学期精力调整）
再次拥有 10 点精力，可更改课程投入档位，重新选择额外活动。其他同 4.1 第二部分。

### 4.4 下半学期结算（期末）
同 4.2，但：
- 课程绩点最终确定（具体数值），并推入 `gpaRecords`。
- 综合整学期点名惩罚、特殊加分等。
- 再次触发 1 个随机事件。
- 期末总结展示最终绩点、属性变化等。
- 身心 ≤0 检查。

### 4.5 学期结束维护
- `currentSemester++`，若为偶数则 `currentYear++`
- 清除短期标记（如 `injured`、`contestWon`）
- 若 `currentSemester > 8` 进入结局判定

## 五、课程数据库

所有课程定义在一个静态 JSON/TS 对象中，结构：

```ts
interface CourseDef {
  id: string;
  name: string;
  type: 'major' | 'politics';
  major: 'EECS' | 'LAW' | 'ALL';   // 适用专业
  attendance: 'none' | 'random' | 'strict' | 'mandatory';
  attendanceRate: number;            // 0, 0.3, 0.8, 1.0
  grading: 'normal' | 'strict' | 'easy' | 'template' | 'fixed' | 'paper' | 'performance' | 'date' | 'favor' | 'fair';
  gradeRange: {                     // 各档位对应的基础绩点区间（未经系数调整）
    skip: [number, number];
    idle: [number, number];
    normal: [number, number];
    serious: [number, number];
    dead: [number, number];
  };
  abilityGainDead: number | [number, number];  // 死磕真实能力增量，或根据条件动态计算
  abilityGainOther: { [key: string]: number }; // 其他档位增量
  mindBodyCost: { [key: string]: number };     // 各档位基础身心消耗
  specialRules?: string;           // 标记特殊逻辑，如 'analog', 'ml', 'research', 'os', 'civil', 'moot', 'writing' 等
  description: string;
  sarcasm: string;
}
```

### 5.1 EECS 专业课（id: c_eecs_c, c_eecs_analog, c_eecs_ml, c_eecs_research, c_eecs_os）
严格按照原始文档的数据（绩点区间、能力增长、身心消耗、特殊规则）填入。例如《操作系统》必须死磕，否则挂科；《科研训练》绩点完全由期末论文（真实能力判定）决定；《机器学习导论》有教师创业事件等。

### 5.2 法学专业课
原始文档有 3 门：民法总论、法律文书写作、模拟法庭。为满足四年 8 个学期，需补充一门课。新增《宪法学》：
- id: c_law_constitution
- 点名：random(0.3)，给分：normal
- 收益：正常学术，死磕真能 +7，身心 -10，无特殊规则。

### 5.3 公共政治课（id: p_moral, p_history, p_mao, p_situation）
严格按文档数据，注意《毛概》不再有混脸熟机制，给分变为普通 standard，绩点上限由 favor 改为正常区间（如正常 3.0-3.5，死磕 3.2-3.7）。

## 七、随机事件系统

### 7.1 事件数据结构
```ts
interface GameEvent {
  id: string;
  priority: number;        // 10: core, 5: satire, 1: daily
  category: 'daily' | 'satire' | 'core';
  trigger: (state: PlayerState, half: 1 | 2) => boolean;
  text: string;
  options?: {
    text: string;
    require?: (state: PlayerState) => boolean;   // 显示条件
    requireDesc?: string;
    effect: (state: PlayerState) => Partial<PlayerState> & { energyDeduct?: number; extraText?: string };
  }[];
  noOptionEffect?: (state: PlayerState) => Partial<PlayerState> & { extraText?: string };
}
```

### 7.2 事件库
必须实现文档中所有事件，包括：
- 通用日常：外卖被偷、恋爱、分手、感冒、受伤、搬宿舍、室友打呼噜、遇到好老师
- 通用讽刺：食堂中毒捂嘴、水课突击小测、综测加分闹剧、学术讲座强制签到、家长电话压力
- 四大核心讽刺：图书馆性骚扰诬陷、竞赛奖金被私吞、保研论文买卖、论文灌水投稿邀请

所有事件的触发条件、选项、效果严格按照文档。选项需求中的金钱/真实能力等要如实判断；精力扣除 `energyDeduct` 在结算时从当前半学期已分配精力中扣除，若不够则扣身心。

### 7.3 触发流程
每个半学期（共16次）必定触发1个事件。先过滤满足条件的，按优先级排序（数字大优先），同优先级随机。触发后记录 id 到 `eventHistory`，部分事件不可重复。

## 八、结局判定

学期计数 > 8 或中途休学/开除时调用 `determineEnding`。

逻辑：
1. 如果 `mindBody <= 0` 或 `expelled` 或 四年内有任何一个学期未选课且未毕业 → 强制休学/延毕结局
2. 计算平均绩点 `avgGPA`，检查政治课是否修完
3. 如果政治课未全部完成或 avgGPA < 1.0 → 延毕/失业
4. 如果 avgGPA >= 3.5 且未被开除 → 保研结局
5. 否则进入考研/就业判定：如果大四有自学行为（设置 `selfstudyHard`）且真实能力 >= 40 → 考研上岸，否则考研失败转就业
6. 就业按真实能力分三档，结合专业给出不同文本

结局文本请严格按照文档，包含辛辣总结。最后显示一句来自手册的总结语。

## 附录：《生存手册》摘要


### 一、 讽刺现实的部分

手册对当前高等教育体系、学校管理、部分教师和学生心态进行了不留情面的批判，主要集中于以下几个方面：

1.  **本科教育“濒临崩溃”**
    *   **学校层面**：学校忙于争创“世界一流”，重科研、轻教学，本科生教学“无利可图”，无人对每个学生的教学质量真正负责。教学培养计划陈旧、不科学，甚至出现过课程依赖顺序颠倒的荒谬情况。
    *   **教师层面**：教授忙于拉经费、发论文，无心教学。部分课程因“因人设课”而存在，为了不让某些教师下岗，内容沦为“活化石”。部分老师水平堪忧，只念PPT，甚至脱离科研生产一线。
    *   **培养模式**：将学生当成“廉价的零件”在流水线上生产，用统一的“教学培养计划”强制应用在所有人身上，无视个体的差异性。这是一种“温水煮青蛙”式的教育，让学生在不自觉中失去竞争力。

2.  **对“分数”与“成功”定义的讽刺**
    *   **GPA的异化**：成绩单是“苍白”的，只在毕业前有用，毕业后便“一文不值”。学校用分数作为单一评价标准，但高分绝不代表真才实学。学生为分数而学，陷入恶性竞争，忽视了能力培养。
    *   **虚假的“管理才能”**：许多学生在专业上无建树，便以“适合做管理”为借口。手册讽刺这种想法为“一厢情愿的错觉”，并指出通往成功管理者的道路比做技术精英更为艰辛。

3.  **对师生关系中“权力不对等”与学术圈乱象的讽刺**
    *   **教师的“权力”**：老师可以因为学生不去上课而视为“奇耻大辱”，想出各种点名方法。学生为了成绩需要“混脸熟”、“搞好关系”，甚至在一门设计课上，老师可以轻易地挑出“莫须有”的问题来贬低学生的工作。
    *   **学术圈的腐败与浮躁**：国内学术圈存在“恶性竞争、派系斗争、造假、剽窃”等不端行为。大量的科研项目是空洞的“文秘工作”，而非真正的研究。所谓的“核心期刊”和“国际会议”成为圈钱工具，存在大量的灌水、买卖论文现象。

4.  **对学生群体“悲壮”与“麻木”的讽刺**
    *   **悲壮的学习方式**：学生像苦行僧一样在图书馆刷题，钻研解题技巧，却只是为了应付考试。这种低效的努力只是“把时间浪费在毫无意义的事情上”，无助于真正的知识积累和国家发展。
    *   **甘于平庸与“被安排的命运”**：许多学生安于学校规划的“平稳中庸”的生活，把自己的青春“放在保险箱里让他随着时间而凋谢”，不敢或不愿去思考更值得做的事情。
    *   **幼稚的出国动机**：把出国当成逃避现状或跟风的手段，没有经过理性思考。手册讽刺地指出，如果一个人在国内都无法成功，凭什么奢望出国后就能“摇身一变”成为胜者。

### 二、 应该做什么（宏观层面）

手册给出了大量具体技巧，但其核心的**宏观行动指南**可以总结为以下几个层面：

1.  **重塑核心观念：从“被动接受到主动思考”**
    *   **摆脱“高考思维”**：大学不再是唯分数论英雄的线性竞争。要清醒认识到，分数只是工具，不是目的。大学四年留下的是你的人生，而不是一串毕业即失效的数字。
    *   **从“被动”到“主动”**：不要为了上课而上课，要问自己“为什么要做这件事”。如果你能通过更高效的自学掌握知识，如果课程本身质量低下，那么“逃课”去做更有价值的事情是完全合理的选择。
    *   **打破“悲壮”的自我感动**：学习最需要的不是“毅力”，而是“对无限未知的渴求”。有效率的学习应以最高效率获取知识，为创造服务。不要沉迷于刷题带来的虚假充实感。

2.  **设定个人战略：明确目标与个人价值**
    *   **认清自己的“身价”**：要对自己未来的价值有清醒的期望。不要为了蝇头小利（如低薪兼职）浪费宝贵的成长时间，而应将时间投资于提升自己未来“高价值劳动”的能力。
    *   **正确“浪费”时间**：把大学四年视为一笔巨大的时间投资。拒绝平庸的生活，反复问自己：“今天的时间过得有价值吗？”多做对整个人生（长期）有正面影响的事，少做或不做只有短期收益（如打游戏）的事。
    *   **尽早规划出路**：在大一就应开始通过多种渠道（教师、学长、企业宣讲）深入了解自己的专业，并根据自身兴趣和长处，想清楚本科毕业后要做什么（保研、出国、就业、转行），越早想清楚，越能提前布局。

3.  **采取具体行动：高效学习与提升自我**
    *   **对课程进行“独立批判”**：面对学校安排的水课、烂课，不必投入过多精力。学会“突击备考”、“理性挂科”和“明智选课”（如选给分好、作业少的老师），把省下来的时间用在刀刃上。
    *   **重新定义“学习”**：课堂学习效率低下，真正有用的知识来自高质量的教科书、英文文献和互联网资源。对于立志于研究的同学，应尽早进入实验室，但必须做**原创性**的工作，而不是“打杂”。
    *   **投身“研究”而非“项目”**：对于PRP等项目要保持警惕，学会辨别项目质量。避免参与空洞的、打杂性质的项目。如果不幸参与，应以自我提升为核心，而不是为了项目而项目。
    *   **锻炼可迁移能力**：除了本专业知识，要着力培养**表达能力、信息检索能力、交际能力、抗压能力、独立思考能力和时间管理能力**，这些是未来在社会立足的根本。
    *   **正确利用学校政策**：深刻理解并利用学校的选课、重修、成绩覆盖、免听等政策，实现个人效率最大化，而不是被政策所束缚。

**总结来说，手册的核心思想是：**

> **大学教育的失败是系统性的，个体必须成为自己命运的“产品经理”。** 不要依赖学校为你塑形，而要主动摆脱失败的思维方式，用批判的眼光审视一切，将宝贵的时间和精力投入到真正能提升长期价值的领域中，走出一条属于自己的、独一无二的道路。
